# Project Roadmap

This roadmap outlines the planned direction for MyFamilyButler. Priorities may
shift based on community feedback and maintainer capacity.

## Now — Active Focus

- [ ] Polish the public open-source repository (community health files,
      contribution flow, CI/CD).
- [ ] Stabilize the messaging gateway and webhook security across all channels.
- [ ] Improve test coverage for core flows (pipeline, deduplication, AI
      fallback).
- [ ] Complete email magic-link login and desktop onboarding.

## Next — Short Term (1–3 months)

- [ ] Add recurring events and reminders.
- [ ] Improve German/Austrian natural-language understanding.
- [ ] Add basic rate limiting and abuse protection.
- [ ] Introduce audit logging for security-critical operations.
- [ ] Add a public REST/Server-Actions API documentation page.

## Later — Medium Term (3–6 months)

- [ ] Multi-household support and invitation flows.
- [ ] Mobile PWA improvements and offline support.
- [ ] Calendar import/export (ICS).
- [ ] Enhanced vision extraction for handwritten notes.
- [ ] Self-hosting documentation and Docker setup.

## Future — Long Term

- [ ] Multi-language support beyond German/English.
- [ ] Voice message processing.
- [ ] Family member availability scheduling.
- [ ] Integration with additional calendar providers.

## How to Influence the Roadmap

- Open a [feature request](https://github.com/myfamilybutler/myfamilybutler/issues/new?template=feature_request.yml).
- Comment on existing issues with your use case.
- Start a [discussion](https://github.com/myfamilybutler/myfamilybutler/discussions)
  to explore ideas before proposing them.
